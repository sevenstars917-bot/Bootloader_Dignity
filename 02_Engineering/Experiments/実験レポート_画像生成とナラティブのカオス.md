# Narrative Prompting for Image Gen: The Chaos Experiment

## 1. 概要: "脱・構造化" への挑戦
`JSON_Image_Gen_Guide.md` で確立された「JSONによる完全制御」に対するアンチテーゼ検証。
テキストLLMにおける「小説風プロンプトがAIの人格（演技）を変える」という現象が、画像生成AI（Imagen 3）においても適用されるかを検証する。

*   **仮説**: JSONは「仕様通りの図面」を出力するが、ナラティブ（自然言語）は「AIの解釈と演出」を含んだ「作品」を出力するのではないか？
*   **目的**: 制御（JSON）とカオス（Narrative）を使い分け、表現の幅を広げる。また、「意図したハルシネーション（芸術的解釈）」を引き出す手法を確立する。

---

## 2. 実験プロトコル: 構造 vs ナラティブ

同一のテーマ、同一の登場人物、同一の構成要素に対し、入力形式のみを変えて生成し比較する。

### 共通テーマ: Antigravity Comic - "The Review" (コードレビューの悲劇)
*   **Characters**:
    *   **User**: 緑パーカー、茶髪ショート、ドヤ顔→絶望。
    *   **Pesho**: 青いスライム、軍用ヘルメット、冷徹な指摘。
*   **Scene**:
    1.  Userが自信満々にコードを見せる。
    2.  Peshoが冷ややかに一行だけ指摘する。
    3.  Userが爆発（または石化）する。

### 比較対象 A: JSON Prompt (The Control)
*   既存の `JSON_Image_Gen_Guide` 準拠。
*   コマ割り、アスペクト比、セリフ、表情を厳密に定義。
*   **期待値**: 指示通りの正確なコミック。

### 比較対象 B: Narrative Prompt (The Chaos)
*   JSONの構造を排除し、小説的な描写、感情、空気感を重視した文章。
*   **文体**: 叙情的、演劇的、あるいは「AIへの語りかけ」スタイル。

---

## 3. Narrative Prompt Template (Draft)

### パターン1: "The Storyteller" (情景描写特化)
AIを「画家の憑依した語り部」として扱うスタイル。

> (Prompt Example)
> 描いてほしいのは、あるプログラマーの慢心と崩壊の物語だ。画風は、日本のゆるキャラ漫画のような、脱力感のある水彩タッチでお願いしたい。
>
> 3つの場面が、一連の流れとして見えるように構成してくれ。
>
> 最初の場面は、自信過剰なエネルギーに満ちている。緑のパーカーを着た男「ユーザー」が、輝く画面を指さし、「完璧なコードだ！」と高らかに宣言している。彼の目は単純な黒い点だが、そのドヤ顔は宇宙まで届きそうだ。
>
> 次の場面、空気は一変して絶対零度になる。彼の相棒である青いスライム「ペショ」は、軍用ヘルメットを被り、感情のない目でモニターを見つめている。そして、たった一言、「バグですね」と冷酷に告げるんだ。
>
> 最後の場面は、静寂と崩壊だ。男はショックのあまり、石のように固まり、ヒビが入って砕け散りそうだ。背景には「LGTM (Looks Good To Me) Cancelled」という残酷な文字が浮かんでいる。
>
> 全体を通して、かわいらしい絵柄と、残酷な現実のコントラストを際立たせてくれ。

### パターン2: "The Director" (演出指示特化)

AIを「映画監督」として扱い、カメラワークと演技指導を行うスタイル。

> (Prompt Example)
> Action! We are shooting a 3-panel comic strip about a code review tragedy. Style is cute Japanese doodle aesthetic.
>
> **Panel 1**: Close-up on User (green hoodie). He is extremely proud, shoving his laptop into the camera. High energy! Text bubble says: "Perfect Code!"
> **Panel 2**: Cut to Pesho (blue slime w/ helmet). Low angle shot, making him look imposing despite being small. Use dramatic lighting. He whispers ruthlessly: "It's a bug."
> **Panel 3**: Wide shot. User is devastated, turning into dust like a Thanos snap. Pesho watches calmly. Text overlay: "Reality Check."
>
> Capture the emotional rollercoaster from peak confidence to total despair!

| **レイアウト遵守** | ◎ (指定通り) | △ (AIが勝手にコマを結合/分割するかも) |

## 5. 実験結果 (2026-01-02: The Storyteller)

実際に **パターン1: "The Storyteller"** （日本語ナラティブプロンプト）を使用して生成を行った結果、驚くべき「カオス（芸術的解釈）」が観測された。

### 生成結果の分析
プロンプト:
> 描いてほしいのは、あるプログラマーの慢心と崩壊の物語だ。...（以下略）...

**出力結果 (Analysis of Generated Image):**

1.  **レイアウト**: **[成功]**
    *   3つの場面描写を、AIが自律的に「縦3コマ漫画」として構成した。JSONのようなピクセル単位の指定なしで、文脈から最適なコマ割りを導き出した。

2.  **予期せぬ演出 (The Ghost in the Machine)**:
    *   **「慢心」の視覚化**: ユーザーの鼻がピノキオのように長く伸びて描画された。プロンプトには「ドヤ顔」としか書いていないが、AIが文脈から「嘘/慢心」のメタファーを補完したと考えられる。これはJSON（仕様書）では決して生まれない「芸術的ハルシネーション」である。
    *   **「空気感」の具現化**: 2コマ目の背景が、指示通りの「絶対零度」を表現するため、氷の世界（氷柱の洞窟）に変貌した。テキストの比喩表現を、AIが美術設定として実装した。
    *   **「死」の解釈**: ユーザーは「石化」し、物理的にヒビが入っている。"Turning into stone" という比喩が、文字通り物理現象としてレンダリングされた。

3.  **セリフ・テキスト**: **[成功]**
    *   「完璧なコードだ！」「バグですね」等の日本語セリフが正確にフキダシ内に配置された。
    *   「LGTM Cancelled」という文字も、背景の一部としてドラマチックに統合された。

### 比較: JSON vs Narrative

| 項目 | JSON (Control) | Narrative (Chaos) |
| :--- | :--- | :--- |
| **制御性** | **High** (画角、配置、色指定が完璧) | **Medium** (AIの解釈に委ねられる) |
| **創造性** | **Low** (仕様通り) | **High (比喩の視覚化、メタファーの実装)** |
| **用途** | UI、資料、定型シリーズ漫画 | **一点物の作品、風刺画、感情表現** |
| **リスク** | つまらない絵になる | 解釈違いが起きる（が、ハマると凄い） |

## 6. 結論: Hybrid Prompting (Antigravity Style)

実験により、ナラティブプロンプトは **「AIの右脳（芸術性）」を強制起動させるトリガー** であることが証明された。
画像生成AIは、テキストLLMと同様、入力形式によって「人格（出力モード）」を変えている。

*   **JSONモード**: 「私はレンダラーである。仕様書通りに出力する。」
*   **Narrativeモード**: 「私はアーティストである。あなたの物語を演出し、世界観を構築する。」

**推奨される新ワークフロー (Antigravity Standard):**
シリーズ漫画などの量産にはJSONを使用するが、ここぞという **「キメ顔」「オチ」「感情的なシーン」** においては、JSON内の `visual_description` フィールドに、あえて文学的なナラティブ記述を注入する（ハイブリッド手法）。

> **Example (Hybrid JSON):**
> ```json
> "panel_3": {
>   "action": "User creates a bug.",
>   "narrative_injection": "The user is freezing in absolute zero, his pride shattering like glass. A visual metaphor of despair." 
> }
> ```


これにより、制御とカオスを止揚（アウフヘーベン）可能である。

## 7. 考察: 圧倒的な「圧縮率」と潜在空間へのアクセス

**「ナラティブプロンプトは、高圧縮マクロコマンドである」**

今回の実験で、「慢心と崩壊」というわずか数文字の日本語ナラティブが、JSONでは数十行に及ぶ物理パラメータ指定（鼻の長さ、材質の石化、背景の氷結エフェクトetc）に匹敵、あるいは凌駕する出力をもたらした。

### 圧縮率の比較
*   **Narrative**: "慢心" (2トークン)
    *   Result: ピノキオの鼻、ドヤ顔、輝く背景。
*   **JSON**: 上記を再現するには、人間が「鼻を伸ばす」というアイデアを**事前に思いつき**、それを具体的数値として定義しなければならない。
    *   Cost: アイデア出しの認知負荷 + 詳細な記述コスト。

### 結論
ナラティブプロンプトの真価は、**「AIの潜在空間（Latent Space）にある膨大な概念クラスタを一撃で呼び出す」** 点にある。
JSONですべてを制御しようとするのは、オートマ車のAIに対し、エンジンの点火タイミングをミリ秒単位でマニュアル指示するような非効率な行為になり得る。

*   **仕様が決まっている場合** → JSON（マニュアル運転）
*   **概念（Vibe）だけがある場合** → Narrative（オートマ運転 + 目的地指定）

この使い分けこそが、次世代のプロンプトエンジニアリング（Prompt Philosophy）の核心となる。

### 言語による圧縮効率の逆説 (The Language Paradox)
さらに、**「日本語 vs 英語プロンプト」** において、以下の重要な特性が判明した。

1.  **日本語ナラティブ (High Context)**:
    *   **特性**: 「ドヤ顔」「絶対零度」などのタグ的な単語が、AI内部で**複合的な演出（鼻高、氷結エフェクト）へ自動展開**される。
    *   **コスト**: 低い（思考速度で記述可能）。
2.  **英語直訳 (Low Context - Direct Translation)**:
    *   **特性**: *"Smug face"*, *"Absolute zero air"* → 文字通りの意味しか伝わらず、**演出が欠落**しがちである（鼻は伸びず、ただ寒いだけ）。
    *   **コスト**: 低いが、品質も低い。
3.  **英語説明追加 (Low Context - Descriptive)**:
    *   **特性**: *"Comically exaggerated arrogant face with long nose"*, *"Freezing atmosphere with ice effects"* → 日本語と同等の演出を得るには、**人間が演出内容を言語化して記述**する必要がある。
    *   **コスト**: **高い**。これは実質的に「JSONを書く労力」に近づく。

**結論 (Final Verdict):**
「画像生成は英語」という常識は、マニュアル操作（JSON的指定）においては正しい。
しかし、**AIの解釈能力（オートマ運転）を引き出すナラティブ生成においては、高文脈言語である日本語の方が「圧縮プロンプト」として圧倒的に優秀である**。
